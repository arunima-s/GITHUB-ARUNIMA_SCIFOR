HTML CSS Basic Assignment 1

1. What is HTML?

HTML (HyperText Markup Language) is the standard markup language used to create web pages. It defines the structure of a webpage by using a series of elements or tags that browsers interpret and render for users. HTML consists of text, images, links, and other types of content.

2. Are the HTML tags and elements the same thing?

No, HTML tags and HTML elements are not the same:

  • Tag: The text enclosed in angle brackets, such as <p>, <h1>, or <div>. Tags are used to mark up the start and end of elements.
  • Element: The entire structure, including the opening tag, content, and closing tag. For example, <p>This is a paragraph</p> is an element.

3. What are the tags and attributes in HTML?

Tags are the building blocks of HTML. They are used to define HTML elements and usually consist of an opening tag and a closing tag (like <h1></h1>). Some tags are self-closing (like <img />).

Attributes provide additional information about an element and are included inside the opening tag. For example: <img src="image.jpg" alt="image description">.

4. What are void elements in HTML?

Void elements are HTML elements that do not require a closing tag. These are self-closing and do not have content between them. Examples include:

  • <img /> (for images)
  • <br /> (for line breaks)
  • <hr /> (for horizontal lines)
  • <input /> (for form inputs)

5. What is the feature of HTML?

Some key features of HTML include:

  • Simplicity: HTML is easy to learn and use, even for beginners.
  • Structure: It provides a standardized way to structure content on the web.
  • Compatibility: HTML works across all modern browsers.
  • Multimedia Support: HTML allows for embedding multimedia elements like images, audio, and video.
  • Hyperlinks: HTML supports links that can connect different web pages.

6. Difference between HTML vs XHTML?

HTML: More lenient and forgiving about errors in syntax, with less strict rules about writing tags and attributes.

XHTML (Extensible HyperText Markup Language): A stricter and more standardized version of HTML, where all tags must be properly closed, attributes must be in lowercase, and documents must be well-formed (XML-based).
